## 정의된 정책 (상세)

### 1. 오행 5종 구성 (`ELEM_CFG`)
| 오행 | 라벨 | 이모지 | 두들 | 대표 맛 계열 |
| --- | --- | --- | --- | --- |
| 목 | 목(木) | 🌿 | DoodleFlower | 신맛/채소 |
| 화 | 화(火) | 🔥 | DoodleLightning | 고기/쓴맛 |
| 토 | 토(土) | 🍯 | DoodleCandy | 단맛/탄수 |
| 금 | 금(金) | 💎 | DoodleCrystal | 매운맛 |
| 수 | 수(水) | 💧 | DoodleMoon | 짠맛/국물 |

### 2. 음식 → 오행 분류 규칙 (`classifyFood`, 키워드 우선순위 순)
정규식으로 입력 텍스트를 매칭, **위에서부터 먼저 걸리는 것**으로 분류:
1. **화(火)** ← `고기|삼겹|치킨|소고기|돼지|닭|생선|회|육`
2. **목(木)** ← `채소|샐러드|나물|김치|상추|브로콜리|시금치|토마토`
3. **수(水)** ← `국|탕|찌개|라면|된장|소금|짠|미역|미소`
4. **토(土)** ← `달달|케이크|초콜릿|아이스크림|설탕|밥|떡|빵|과자|쿠키`
5. **금(金)** ← `매운|고추|마늘|양파|생강|후추|겨자`
6. **fallback**: 어디에도 안 걸리면 `첫 글자 charCode % 5`로 [목,화,토,금,수] 중 결정 (결정적, 항상 같은 입력→같은 오행)

> ⚠️ 주의: 규칙이 순서 의존적. 예) "된장찌개"는 화 키워드 없고 '찌개'가 수에 걸려 **수**. "닭갈비"는 '닭'이 먼저라 **화**(매운맛이어도 금 아님).

### 3. 기본 제안 칩 (`DEFAULTS`)
처음부터 등록 대기열(pending)에 담겨 있는 4개: `고기류`(화), `채소`(목), `국물`(수), `달달한 것`(토). 등록하면 다시 이 4개로 리셋됨.

### 4. 등록/카운트 로직
- 입력→`분석` 버튼/Enter→700ms 가짜 로딩(🔮)→분류 후 pending에 칩 추가.
- `없애기` 토글 시 pending 칩 개별 삭제 가능(× 표시).
- `등록하기`→pending 전체를 `counts`에 +1씩 누적, 스티커로 시각화, pending은 기본 4칩으로 리셋.
- `lastElem` = 마지막 등록 묶음의 맨 끝 칩 오행 → 반응 멘트 기준.

### 5. 반응 멘트 정책 (`REACTIONS` + `getReaction`)
오행별 누적 카운트(`counts[elem]`)에 따라 3단계:
- **부족(lack)**: `count === 0` — 해당 오행 결핍 증상 + 맛 추천 (예: 수 부족 → "몸이 건조해지고 있어. 물 마시고…")
- **정상(normal)**: `1 ≤ count ≤ 2` — 충전 중 긍정 멘트
- **과다(excess)**: `count >= 3` — 과잉 경고 (예: 토 과다 → "단 거 너무 많이 먹었는데… 위장아 괜찮니?")

전체 멘트 원문은 백로그의 `FoodLogger.tsx` `REACTIONS` 객체 참조.

### 6. 오행 게이지 + 용신 목표선 (`YONGSHIN_GOAL`)
- 각 오행 막대 = `counts[e] / total * 100` %.
- **용신 목표 마커**: `YONGSHIN_GOAL = { 수: 40 }` — 수(水) 막대 위 40% 지점에 "목표" 세로선 표시. (현재 mock은 수만 용신으로 하드코딩 — 실제 연동 시 유저 사주의 용신/희신 오행으로 대체해야 함)
- `total === 0`이면 게이지 숨기고 "먹은 거 추가하면 오행 게이지가 쌓여 ✨" 안내.

### 7. 주간 밸런스 리포트 (`WEEKLY_MOCK`) — 구독자 전용 게이트
- mock 데이터: `{ 목:5, 화:9, 토:7, 금:3, 수:2 }`. 최다/최부족 오행 자동 계산해 카드로 표시.
- 막대 차트 위에 `backdrop-blur` + 🔒 **"구독자 전용"** 오버레이 + "구독하기 · ₩2,900/월" CTA.
- 즉, **주간 리포트는 유료(구독) 기능**으로 설계됨. 일일 트래커(입력/게이지/반응)는 무료.

## 원본 기획 지침 (APP-UI-PLAN.md `### B. 오행 밸런스 트래커 상세`에서 이전, UI 포함)

> 라이브 문서(APP-UI-PLAN.md)에서 제거하고 이리로 합침. 위의 구현 정책(1~7)이 이 기획을 실제로 옮긴 것.

**기본 기능:**
- 오늘 먹은 것 텍스트 입력
- AI가 오행 분류 (목/화/토/금/수)
- 오행별 게이지 (5개 막대)
- 호들갑 떠는 반응 멘트
- 부족/과잉 오행에 따른 행동 제안

**고도화:**

빠른 선택지 제공:
```
[🥩 고기류] [🥦 채소] [🍜 국물] [🍰 달달한 것] [직접 입력]
```

오행별 호들갑 멘트 풀:
- 화(火) 과잉: "헐, 불 뿜겠다. 오늘 뭔가 시작하기 엄청 좋은 날이네"
- 화(火) 과잉 2: "제육볶음 먹었다고? 불의 기운이 넘쳐. 얼른 물 한 잔 들이켜!"
- 수(水) 부족: "몸이 건조해지고 있어. 물 마시고, 조용한 곳 잠깐 가봐"
- 목(木) 과잉: "성장 욕구 폭발 상태야. 새로운 거 시작하고 싶어 미치겠지?"
- 금(金) 부족: "우유부단해지기 딱 좋은 상태. 결정 미루지 마"
- 토(土) 과잉: "안정감 넘쳐. 오늘은 그냥 쉬어도 돼"

스티커 누적 시각화:
```
  오늘 먹은 것들:
  🔥🔥🔥🔥  (제육볶음, 라면, 떡볶이, 커피)
  🌳🌳       (샐러드, 사과)
  ⛰️⛰️       (밥, 감자)
  💧          (물)
```

용신 기반 목표 게이지:
- 내 용신이 수(水)면 수(水) 게이지에 목표선 표시
- "여기까지 채워봐!" 표시

주간 밸런스 리포트 (구독자만):
- 일주일 오행 집계
- "이번 주 화(火) 2위. 수(水) 계속 부족해"
- 다음 주 추천 식단 힌트
