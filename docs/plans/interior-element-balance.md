## 0. 한 줄 컨셉

사용자가 미니홈피에 배치한 **모든 아이템(소품·캐릭터·스킨)의 오행을 합산**해
**"내 방의 기운"**을 보여주고, 그 기운이 **내 사주 전체 오행 분포와 어떻게 어울리는지**까지
해석해주는 기능. 단순 꾸미기를 넘어 "이 방이 나에게 좋은 기운을 주는가?"라는
사주 서비스만의 의미를 부여하고, 부족한 오행 아이템 추천으로 구매를 유도한다.

---

## 1. 데이터 기반 (현황)

| 대상 | 오행 출처 | 상태 |
|---|---|---|
| 소품(스티커) | `DoodleEntry.element` (`목/화/토/금/수/무/전체`) — `doodle-categories.ts` | 옵셔널 → **필수 승격 예정** |
| 캐릭터 | 일주키 → `stemElement` (`ilju-calc.ts`) | 있음 (조회) |
| 스킨 | 신규 부여 (색상 매핑) | **추가 예정** |

- 배치 데이터 `RoomData = { stickers, chars, charPos, skinId }` — 스티커는 `name`만,
  캐릭터는 `key`(일주키)만 보유 → 오행은 조회로 역산.
- **신규 유틸**: `getStickerElement(name)` — `DOODLE_CATEGORIES`에서 `name → element`
  룩업 맵을 1회 빌드.

---

## 2. 결정 사항 (확정)

| 항목 | 확정값 |
|---|---|
| `무` 오행 | **집계 제외** (중립 데코, 분모에서도 제외) |
| `전체` 오행 | **5행 균등 분배** (각 행 +0.2) |
| 스티커 가중치 | **×1** |
| 캐릭터 가중치 | **×3** (정체성) |
| 스킨 가중치 | **×2** (배경 전체 존재감) |
| 스킨 오행 | **색상 매핑** (아래 표) |
| 해석 기준 | **사주 전체 오행 분포** (`calcElementDistribution`) |
| `element` 필드 | **옵셔널 → 필수 승격** |

---

## 3. 계산 로직 — "방 오행 점수"

```
방 오행 점수[행] = Σ(스티커 element)     × 1
               + Σ(캐릭터 stemElement)  × 3
               + (스킨 element)          × 2
```

- `무` → 어떤 행에도 가산하지 않음
- `전체` → 5개 행에 각 0.2씩 분배 (합 1)
- 정규화해서 **비율(%)** 산출
- 산출 결과: 방의 **대표 기운**(최대), **부족한 기운**(최소/0), 5행 분포

> 향후(Phase 3): 스티커 `scale`(크기) 비례 가중 검토.

## 4. 스킨 → 오행 색상 매핑 (확정)

오행 표준 색(목=청록, 화=적·분홍, 토=황·베이지, 금=백·회, 수=흑·청) 기준.
`RoomSkin` 타입에 `element: ElemKr` 필드 추가.

| 스킨 id | 이름 | 대표색 | → 오행 |
|---|---|---|---|
| `default` | 기본 | 크림·베이지 | **토** |
| `pink` | 봄 벚꽃 | 분홍 | **화** |
| `blue` | 달빛 하늘 | 하늘색 | **수** |
| `mint` | 민트 숲 | 민트·초록 | **목** |
| `lemon` | 레몬 | 노랑 | **토** |
| `ocean` | 여름 바다 | 청색 | **수** |
| `lavender` | 라벤더 | 연보라 | **화** *(확정)* |

## 5. 해석 엔진 — 사주 분포 비교 (확정)

1. **신규 파생 훅** `useSajuDistribution()`
   - `birthDate`(생시·성별 포함) → `calcSajuPillars()` → `calcElementDistribution()`
   - 사주 8글자 오행 분포(정규화) 반환.
   - 생시 모름 케이스: 시주 제외(`calcSajuPillars`의 `hour: null` 지원).
2. **키 정규화 브릿지** (필수)
   - `calcElementDistribution`은 `"목(木)"` 형태, 방 집계는 `"목"`(`ElemKr`) 형태
     → 변환 매핑 1개 추가.
3. **비교 → 메시지 분기**
   - 사주에 **부족한** 행을 방이 채움 → 보완형(긍정)
   - 사주에 **과한** 행이 방에도 과함 → 균형 권유형
   - 사주를 **상생**하는 행이 방에 풍부 → 길(吉) 강조
4. 결과 → **부족 오행 소품 추천 CTA** (구매 연결)

> ⚠️ 톤: 평가·단정 금지("~이 부족합니다" 금지). 따뜻한 권유체.

## 6. UI/UX — 직관적 표현

- **(A) 인테리어 메인 — "방의 기운" 카드 (신규)**
  `MiniRoom` 아래 삽입. 5색 오행 도트/스택바(기존 `ELEM_RING`/`ELEM_BG_MAP` 색 활용)
  + 대표 기운 헤드라인 + 한 줄 해석. 탭 → 상세 바텀시트.
- **(B) 상세 바텀시트**
  5행 분포 차트(레이더/가로막대) + 아이템별 기여 내역 + 사주 비교 해석 문단
  + 부족 오행 추천 CTA.
- **(C) 보관함/방 편집기 — 오행 뱃지**
  각 소품·캐릭터 카드에 작은 오행 색 뱃지(목🟢 화🔴 토🟡 금⚪ 수🔵)
  → 놓기 전부터 "무슨 기운이 더해지는지" 인지.
- **(D) 편집 중 실시간 미터**
  아이템 배치할 때마다 밸런스 미터 실시간 갱신.

## 7. 단계별 로드맵

**Phase 1 — 기반 + 시각화**
`getStickerElement` 유틸 / 방 오행 집계 함수(가중치 1·3·2) / 스킨 `element` 필드 추가 /
메인 "방의 기운" 카드(도트+%) / 보관함 오행 뱃지. *(해석 없이 분포만)*

**Phase 2 — 해석 + 추천**
`useSajuDistribution()` 훅 + 키 브릿지 / 사주 비교 해석 엔진 / 상세 바텀시트 /
부족 오행 소품 추천 CTA.

**Phase 3 — 심화/게이미피케이션**
`scale` 비례 가중 / "오행 밸런스 달성" 칭호·보상 / 친구 방 기운 비교.
